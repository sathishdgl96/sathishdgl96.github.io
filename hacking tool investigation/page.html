5 ways of compiling
  • 1. statically linked program = All modules are available in a single module [Standalone , no dependency], Capable of executing in any version of the OS
  • gcc - static zap.c

  • 2. Dynamically linked program = use shared library to execute [Making use of existing library routines]. Easy to transmit
  • 3. Programs compiled with debug options = Normally given in a developement environment.
  • gcc -g zap.c -o zap debug
  • Depending on the debugging level, the gcc will give information
  • 4. Stripped Programs
  • Strip is a function that deletes all symbols and exe file will be smaller . If dynamically linked and stripped file are combined then the prgram will be extremely small
  • nm is a command to bring back symbols [TO make unstripped]
  • strip is a cmd to make file stripped
  • 5. Compliation with UPX [ultimate packers for executables]
  • Due the compression methodology follows, the file size be smaller
  • less clues
  • Popular among attackers
  • win32 applications, dos 16-bit exe, dos 32 bit exe

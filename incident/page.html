IR Process:

Involved with three main activites
1. Initial Response
2. Investigation
3. Remediation

1. Initial Response:
  • Assemble the team
  • Reviewing Readily available data
  • Identify the type of incident
  • Asses the impact of the incident
Everything should be documented
2. Investigation:
Answering these questions ... What, Who, How happened ?
Initial Leads -> Indicator of Comprosimed (IOC) -> Deploy IOC -> Identify System of Interest ->Collect Evidences ->Analyse Data
  • 1. Initial Leads = Identify the intial clues (Recognize the type of incident)
      • A good lead must be relevant, detailed, Actionable
  • 2. IOC - Indicators of compromised
      • Some Compromised have been occured in a system = Every contact leaves a trace
      • Defintion:IOC is an artifact observed on a network or os that with high confidence indicates a computer intrusion Eg: Virus Signatures, Ip adrresses, URL, Domain Names
      • Process of documenting in a structured manner
      • Tools: YARA, Mandiant's, mitre's
  • IOC can be collected from individual system or in a network
      • Ways of Collecting IOC
      • 1. Monitoring a network
      • 2. Unnecessary I/O write
      • 3. Geographical Irregularities
      • 4. Anything that can un usual
  • 3. Deploy IOC
    • Collections and reporting should be done automatically
  • 4. Identify System of Interest
    • Getting the hits from the IOC (Above the threshold)
    • 1. Validate - Whether SOI is reliable or not
    • 2. Categorize - SOI should be categorized
    • 3. Priotize - Highest prority Hits should be priotize
  • 5. Collect and preserve Evidences:
    • Collect Additional Evidences to justify the suspicion
      • Investigation Preservation
      • 1. Live response
      • 2. Memory Collection - Main Memory ( Incase of rootkit)
      • 3. Forensic DIsk image - Duplicating Disk And Preserving Evidence
  • 6. Analysing the Data
    • Try to answer the questions (What, How, Who)
    • Malware Analysis
    • Live Response analysis - More opportunity to collect leads
    • Forensic Examination - Disk
3. Remediation:
Legal, political, business actions
Timing is a crucial - Not too early (You will get enough data) , too late ( Damages will be more)
    • Three methods:
    • 1. Postering - Establish a new protocol, exchanging the contact info, designating the responsibilites, scheduling resources, co-ordinating timeline
    • 2. Tactical - Address the current situation and react for it ( Short term remidation) (blocking ip address, informing the customers, changing passwords)
    • 3. Strategy - Similar incident should not happen again (Long term Remediation ) - requires changes in an organization

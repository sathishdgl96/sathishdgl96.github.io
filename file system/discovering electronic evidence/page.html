Approach to convert the data into evidence:
  • 1. Define and understand the objectives
  • 2. Towards Objectives, collect some relavant data
  • 3. Inspect or analyze the data content
  • 4. If necessary go for format convertion and normalization
  • 5. Choose a methos to do analysis
  • 6. Perform analysis
  • 7. Evaluate
These steps are iterative...

1. Objectives:
know the situations and technologies available at the incident spot
What are we looking for
draw conclusion
How long it will take to complete the analysis
What are the resource need
What are the people who are interested in my results [Who need the results & how are they going to process my results]
Leadership - A Team of people will be working in a analysis [This prevents mis understandings and focus]
Objectives should be focused on not proving a negative [Eg: Prove that there are possible for compromisation ]

You must ask only realistic question [Is there any malware present - Non realistic at the beginning; is file present - Realistic ]

Scope of an objective:Should not be too vague [looking at entire hard disk, full inbox]
=> to be particular hard disk , or particular inbox

These must be asked to all the stalkholders involving in a incident

Know your data: Where is your data? In a server, cloud, hard drive
, Laptop, Mobile Device, Are you dealing with firewall data, cloud services, networking devices

Types of evidence:
1. Operating system evidences -

1. NTFS, FAT, HFS[Info about active, deleted files, unallocated space, or file slack, or parition table]
2. State Information - Threads and Processes information / Port state info
3. OS logs: - Log information
4. Os Specific data sources

2. Applications:
1. Browser cache
2. Web server logs
3. User preferences
4. Email client files
Usually applications will be uninstalled after crime, but traces will be left

3. User data
1. From doc, emails, source codes
2. From web servers

4. Network/ Instrumentation data:
1. Network flow data
2. firewalls
3. Proxy Servers
